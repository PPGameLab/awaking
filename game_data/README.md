# Game Data

**Игровые данные и механики (runtime).**

## Назначение

`game_data/` содержит данные, которые **реально использует игра (runtime)**:
- Механики
- Системы
- Баланс
- Структура прогрессии
- Конфигурация игровых систем

## Правило разделения

**Если файл нужен игре (runtime) → `game_data/`**  
**Если это "описание/план" → `design/`**

### Примеры

**`game_data/`** (runtime):
- `attributes.mvp.json` — система характеристик (игра читает)
- `magic_system.mvp.json` — система магии (игра читает)
- `game_mechanics.mvp.json` — базовая механика (игра читает)

**`design/`** (спеки/планы):
- Спеки будущих систем
- Черновики механик
- Планы баланса
- Описания дизайна

## Важное правило

**"Лор" и "Дизайн" не смешиваем.**

- `lore/` — мир, королевства, понятия, канон
- `game_data/` — механики, системы, баланс, структура прогрессии (runtime)
- `design/` — спеки и черновики систем (не обязательно runtime)

## Структура

- `magic_system.mvp.json` — система магии для игрока (MVP версия)
- `attributes.mvp.json` — система характеристик и параметров юнитов (MVP версия)
- `game_mechanics.mvp.json` — базовая механика игры (MVP версия)
- `map_structure.mvp.json` — структура карты мира (MVP версия)
- `map_config.mvp.json` — конфигурация генерации карты (MVP версия)
