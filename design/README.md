# Design

**Спеки и черновики систем (не обязательно runtime).**

## Назначение

`design/` содержит описания и планы систем, которые не обязательно используются игрой напрямую:
- Спеки систем
- Черновики механик
- Планы разработки
- Описания дизайна

## Правило разделения

**Если файл нужен игре (runtime) → `game_data/`**  
**Если это "описание/план" → `design/`**

### Примеры

**`game_data/`** (runtime):
- `attributes.mvp.json` — система характеристик (игра читает)
- `magic_system.mvp.json` — система магии (игра читает)
- `game_mechanics.mvp.json` — базовая механика (игра читает)

**`design/`** (спеки/планы):
- Спеки будущих систем
- Черновики механик
- Планы баланса
- Описания дизайна

## Структура

```
design/
├── README.md           # Этот файл
└── [спеки и черновики...]
```
