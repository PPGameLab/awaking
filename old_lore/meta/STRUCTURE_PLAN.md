# План структуры данных лора

## Разделение сущностей

### Королевства
- **Путь:** `lore/tables/kingdoms.*`
- **Формат:** JSON или Markdown таблицы
- **Содержание:** Структурные данные о королевствах (ID, название, атрибуты, расположение)

### Регионы/Зоны
- **Путь:** `lore/tables/regions.*` или `lore/canon/regions/*`
- **Формат:** JSON или Markdown
- **Содержание:** 
  - Центральная зона
  - Другие регионы/зоны
  - Нейтральные территории

## Проблема в legacy данных

⚠ **Важно:** В `lore/drafts/legacy/kingdoms_legacy.json` смешаны королевства и регионы:
- Королевства (8 штук) — должны быть в `tables/kingdoms.*`
- `center-zone` (регион) — должен быть в `tables/regions.*` или `canon/regions/*`

**Это будет ломать логику**, если не разделить при переносе в актуальные данные.

## При переносе из legacy

При переносе данных из `lore/drafts/legacy/` в актуальные структуры:
1. Королевства → `lore/tables/kingdoms.*`
2. Центральная зона и другие регионы → `lore/tables/regions.*` или `lore/canon/regions/*`
