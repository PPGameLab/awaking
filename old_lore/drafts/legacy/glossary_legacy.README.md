# glossary_legacy.json

⚠ **LEGACY DATA DUMP**

Содержит смешанные статусы (CANON-DRAFT/PROPOSAL) и разные типы сущностей.
Не является источником истины.

Актуальные термины должны жить в `lore/glossary/` (1 файл = 1 термин).
Вопросы и предложения — в `lore/meta/open_questions.md` и/или `lore/drafts/`.

---

## ✅ Правила для будущего глоссария

**Глоссарий = только определения.**

В нём не должно быть:
- ❌ "story hooks"
- ❌ "open questions" внутри записи (вынести отдельно)
- ❌ списков "relevantKingdoms" (это уже связи и таблицы)

**Структура:**
- Термин → `lore/glossary/<term>.md`
- Связи термин↔королевство → таблица `lore/tables/kingdom_term_links.csv`
- Вопросы → `lore/meta/open_questions.md`

---

## ⚠️ Особое замечание: статусы

**CANON-DRAFT запрещён** (см. `lore/meta/RULES.md` — правило для всего проекта).

Разрешённые статусы:
- **CANON** — зафиксировано
- **DRAFT** — в работе
- **IDEA** — гипотеза

**Статус только в метаданных, а не внутри текста.**
